# Antigravity Tools Project Documentation

## 1. 项目基础信息
- **项目名称**: Antigravity Tools
- **项目类型**: Tauri v2 桌面应用 (Rust + React)
- **核心功能**: AI 账号管理与 API 协议反代 (Claude/Gemini -> OpenAI)

## 2. 技术栈 (Stack)
### 前端 (Frontend)
- **框架**: React 19
- **构建工具**: Vite 7
- **语言**: TypeScript (~5.8.3)
- **样式**: TailwindCSS 3 + DaisyUI 5
- **状态管理**: Zustand
- **路由**: React Router 7
- **国际化**: i18next
- **图标**: Lucide React

### 后端 (Backend / Core)
- **运行时**: Rust (Tauri v2)
- **Web 框架**: Axum (内嵌反代服务器)
- **异步运行时**: Tokio
- **数据库**: SQLite (rusqlite)
- **以及**: commands, modules, proxy (mappers/handlers)

### 基础设施
- **打包**: Tauri CLI v2
- **容器化**: Docker (headless mode)

## 3. 目录结构规范
```
/
├── .github/            # GitHub Actions (CI/CD)
├── src/                # 前端源码
│   ├── components/     # UI 组件
│   ├── pages/          # 路由页面
│   ├── stores/         # Zustand 状态
│   ├── services/       # API/IPC 服务
│   ├── App.tsx         # 根组件
│   └── main.tsx        # 入口
├── src-tauri/          # Rust 后端源码
│   ├── src/
│   │   ├── commands/   # Tauri Commands
│   │   ├── modules/    # 业务逻辑
│   │   ├── proxy/      # API 反代核心
│   │   ├── models/     # 数据模型
│   │   └── lib.rs      #以此为库入口
│   ├── Cargo.toml      # Rust 依赖
│   └── tauri.conf.json # Tauri 配置
└── docker/             # Docker 构建文件
```

## 4. 文件规范
- **单文件限制**: 最大 800 行, 超过需拆分
- **代码风格**:
  - Rust: `cargo fmt`
  - TS/JS: ESLint + Prettier
- **组件命名**: PascalCase (如 `AccountList.tsx`)
- **Hook 命名**: camelCase (如 `useAccountStore.ts`)
